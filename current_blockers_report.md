# 프로젝트 현황 및 주요 블로커 (Blocker) 리포트

전체 저장소의 Git 상태와 최근 개발일지, 코드 변경 사항을 순회하며 파악된 핵심 블로커와 위험 요소입니다.

## 1. 치명적 블로커: 모델 추론부 Feature Shape Mismatch 및 제로(0) 패딩 편법
가장 심각한 기술적 부채이자 블로커는 SG_proj_001(PolySim)의 점착력 예측 모델에서 발생한 차원 불일치(Expected: 84, Got: 85) 에러에 대한 핫픽스 처리입니다.
* 문제 상황: 추론 엔진(engine.py) 단에서 [X = X[feature_names]]를 강제 적용하여 누락된 컬럼을 일괄 0으로 채우는 방식이 적용되어 있습니다.
* 위험성: 강제로 0을 채우는 행위는 화학 도메인 관점에서 해당 모노머나 첨가제가 "0% 투입되었다"는 확정적 편향을 부여하게 됩니다. 이는 트리기반(XGBoost) 모델의 분기 트리에서 완전히 잘못된 노드를 타게 만들어 추론 결과값을 붕괴시킵니다 (성능이 0.17에서 0.24로 미세 상승했다고 로그에 적혀있으나 여전히 사용 불가 수준의 R2 스코어입니다).
* 해결 필요: 편법 핫픽스를 제거하고, 훈련용 데이터 파서(parse_monomers_v3.py)와 인퍼런스 단의 데이터 추출 로직 간의 피처(Feature) 스키마를 근본적으로 일치시켜 새롭게 모델을 재학습해야 합니다.

## 2. 형상 관리 블로커: 대규모 아키텍처 변동 사항 미커밋(Untracked) 방치
Git 상태 조회 결과, 001 및 014 레포지토리에 매우 방대한 양의 핵심 파일들이 커밋되지 않고 방치되어 있습니다.
* SG_proj_001: 새로운 딥러닝 역설계 아키텍처인 CVAE 모델 파일들(cvae_inverse.pt, cvae_scaling.pt)과 다목적 최적화(NSGA-II) 로직이 추가되었으나 버전 관리에 등록되지 않았습니다.
* SG_proj_014: 001의 신규 엔진을 호출하기 위한 API 엔드포인트(/inverse_cvae, /inverse_nsga2) 라우팅 파일들이 수정되었으나 커밋 대기 상태입니다.
* 조치 필요: 현재 상태에서 서버 장애나 롤백 상황이 발생할 경우 새로 도입된 0.1초 다이렉트 역설계 아키텍처가 전부 유실될 위험이 있습니다. 즉시 E2E 테스트를 거쳐 안전하게 커밋하고 형상을 고정해야 합니다.

## 3. 통신 규격 엇갈림 (UI 프론트엔드 API 미갱신)
방금 015의 임시 시연 UI를 기존 013 역설계 게이트웨이 루프로 연동해 두었는데, 014 오케스트레이터의 최신 개발일지를 보면 CVAE 기반의 초고속 역설계 엔드포인트가 신설되었습니다. 현재 UI는 낡은 알고리즘 엔드포인트에 묶여있을 가능성이 높으므로, 이 신규 딥러닝 역설계 API로 프론트엔드 연동부를 전환(Migration)해야 하는 연결 지연 블로커가 존재합니다.
